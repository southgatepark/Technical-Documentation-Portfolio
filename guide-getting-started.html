Get Your API Key

‚è±Ô∏è 3 minutes

API keys authenticate your requests and track your usage. Each key is tied to your account and can be regenerated or revoked at any time from the dashboard.

Generate Your First API Key

  • Navigate to Settings ‚Üí API Keys in your dashboard
  • Click "Create New Key"
  • Give your key a descriptive name (e.g., "Development" or "Production")
  • Select the appropriate permissions (start with "Read/Write" for testing)
  • Copy and securely store your API key
‚ö†Ô∏è Security Warning
Your API key is shown only once. Store it securely‚Äîtreat it like a password. Never commit API keys to version control or share them publicly.

Best Practices for API Keys

Store your API key as an environment variable:

Make Your First API Request

‚è±Ô∏è 5 minutes

Now that you have your API key, let's make your first request. We'll start with a simple GET request to retrieve your account information.

Authentication

All API requests must include your API key in the Authorization header as a Bearer token:

HTTP 
Authorization: Bearer YOUR_API_KEY

Your First Request

Let's fetch your account details. Choose your preferred language below:

cURL 
curl https://api.example.com/v1/account \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json"
Python 
import requests
import os

api_key = os.getenv('API_KEY')
headers = {
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.example.com/v1/account',
    headers=headers
)

if response.status_code == 200:
    account = response.json()
    print(f"Account ID: {account['id']}")
else:
    print(f"Error: {response.status_code}")
JavaScript 
const fetch = require('node-fetch');

async function getAccount() {
  const response = await fetch('https://api.example.com/v1/account', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${process.env.API_KEY}`,
      'Content-Type': 'application/json'
    }
  });

  if (response.ok) {
    const account = await response.json();
    console.log(`Account ID: ${account.id}`);
  } else {
    console.error(`Error: ${response.status}`);
  }
}

getAccount();
Go 
package main

import (
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "os"
)

func main() {
    apiKey := os.Getenv("API_KEY")
    
    req, _ := http.NewRequest("GET", "https://api.example.com/v1/account", nil)
    req.Header.Set("Authorization", "Bearer "+apiKey)
    req.Header.Set("Content-Type", "application/json")
    
    client := &http.Client{}
    resp, err := client.Do(req)
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()
    
    body, _ := io.ReadAll(resp.Body)
    fmt.Println(string(body))
}

Handle API Responses

‚è±Ô∏è 5 minutes

Understanding how to handle API responses is crucial for building robust integrations. Let's explore successful responses, error handling, and rate limits.

Successful Response Format

All successful responses return JSON with a consistent structure:

JSON 
{
  "id": "acct_1MqBy...",
  "object": "account",
  "email": "developer@example.com",
  "created": 1677721234,
  "settings": {
    "timezone": "America/Los_Angeles",
    "currency": "USD"
  }
}

Error Handling

When something goes wrong, the API returns an error response with details:

JSON 
{
  "error": {
    "type": "authentication_error",
    "message": "Invalid API key provided",
    "code": "invalid_api_key",
    "doc_url": "https://docs.example.com/errors/invalid-api-key"
  }
}

Common HTTP Status Codes

  • 200 OK - Request succeeded
  • 400 Bad Request - Invalid request parameters
  • 401 Unauthorized - Authentication failed
  • 404 Not Found - Resource doesn't exist
  • 429 Too Many Requests - Rate limit exceeded
  • 500 Server Error - Something went wrong on our end
üí° Rate Limits
Free tier accounts are limited to 100 requests per minute. Check the X-RateLimit-Remaining header to see how many requests you have left.
