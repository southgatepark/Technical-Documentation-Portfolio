Command

devctl deploy

Deploy your application to the specified environment. This command builds your application, runs tests, and deploys to the target infrastructure with zero-downtime rollout.

Syntax
devctl deploy <environment> [flags]

Flags

--build-arg -b string

Set build-time variables. Can be used multiple times to set multiple variables.

Default: none
--config -c string

Path to configuration file. Overrides default config locations.

Default: ./devctl.yaml
--dry-run boolean

Simulate the deployment without making actual changes. Useful for validation.

Default: false
--force -f boolean

Skip confirmation prompts and health checks. Use with caution in production.

Default: false
--tag -t string Required

Docker image tag or Git commit SHA to deploy. Must be a valid container image tag.

--timeout duration

Maximum time to wait for deployment to complete. Accepts values like "30s", "5m", "1h".

Default: 10m
--verbose -v boolean

Enable verbose output with detailed deployment logs and progress information.

Default: false

Examples

Deploy to production with specific tag
Deploy version v2.5.0 to the production environment 
$ devctl deploy production --tag v2.5.0
Dry run deployment
Validate your deployment configuration without actually deploying 
$ devctl deploy staging --tag latest --dry-run
# Output shows what would happen without making changes
Deploy with build arguments
Pass multiple build-time variables for the deployment 
$ devctl deploy development \
  --tag dev-latest \
  --build-arg NODE_ENV=development \
  --build-arg API_URL=https://dev-api.example.com \
  --verbose
Force deployment with custom timeout
Deploy to staging, skipping confirmations with extended timeout 
$ devctl deploy staging --tag v2.4.1 --force --timeout 15m

Always test deployments in a staging environment first. Use --dry-run to validate your deployment configuration. For production deployments, avoid using --force unless absolutely necessary.

Command

devctl config

Manage configuration settings for your environments and applications. View, set, and delete configuration values across all environments.

Syntax
devctl config <subcommand> [flags]

Subcommands

get
Retrieve configuration value
set
Set configuration value
list
List all configuration keys
delete
Remove configuration key

Examples

Set configuration value
$ devctl config set production.api.url https://api.example.com
Get configuration value
$ devctl config get production.api.url
# Output: https://api.example.com
List all configuration
$ devctl config list --environment production
Command

devctl logs

Stream or retrieve logs from your deployed applications. Supports filtering, following live logs, and exporting to files.

Syntax
devctl logs <environment> [flags]

Examples

Follow live logs
$ devctl logs production --follow --tail 100
Filter logs by level
$ devctl logs staging --level error --since 1h
