REST API

Payment API Reference

Complete REST API documentation for processing payments, managing transactions, and handling refunds. All endpoints require authentication via API key.

POST /v2/payments

Creates a new payment transaction. This endpoint initiates a payment request and returns a payment object with a unique identifier. The payment can be completed immediately or authorized for later capture.

Authentication Required

This endpoint requires a valid API key passed in the Authorization header as a Bearer token.

Request Body Parameters

Parameter Type Description
amount Required integer Payment amount in smallest currency unit (e.g., cents for USD)
currency Required string Three-letter ISO currency code (e.g., "USD", "EUR", "GBP")
payment_method string Payment method identifier or object
customer_id string ID of existing customer for this payment
description string Arbitrary description for the payment
metadata object Set of key-value pairs for storing additional information
capture_method enum Either "automatic" or "manual". Defaults to "automatic"
Request Example curl 
curl -X POST https://api.payments.example/v2/payments \
  -H "Authorization: Bearer sk_test_4eC39HqLyjWDarjtT1zdp7dc" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 2500,
    "currency": "usd",
    "payment_method": "pm_1234567890",
    "customer_id": "cus_ABC123",
    "description": "Subscription payment",
    "metadata": {
      "order_id": "ord_987654",
      "product": "Premium Plan"
    }
  }'

Response

200 Success

  "id": "pay_1MqBx72eZvKYlo2C8zGz5HqP",
  "object": "payment",
  "amount": 2500,
  "currency": "usd",

  "customer_id": "cus_ABC123",
  "payment_method": "pm_1234567890",
  "description": "Subscription payment",
  "created": 1677721234,

  "metadata": {
    "order_id": "ord_987654",
    "product": "Premium Plan"
  },
  "receipt_url": "https://payments.example/receipts/pay_1MqBx72eZ..."
}

Error Responses

Status Code Description
400 Invalid request parameters or missing required fields
401 Authentication failed or API key is invalid
404 Payment method or customer not found
402 Payment failed (insufficient funds, declined card, etc.)
Error Response Example json 
{
  "error": {
    "type": "card_error",
    "code": "card_declined",
    "message": "Your card was declined",
    "param": "payment_method",
    "decline_code": "insufficient_funds"
  }
}
GET /v2/payments/{id}

Retrieves the details of an existing payment. Supply the unique payment ID from either a payment creation request or the payment list.

Path Parameters

Parameter Type Description
id Required string Unique identifier for the payment
Request Example curl 
curl https://api.payments.example/v2/payments/pay_1MqBx72eZvKYlo2C8zGz5HqP \
  -H "Authorization: Bearer sk_test_4eC39HqLyjWDarjtT1zdp7dc"
200 Success - Returns the payment object
